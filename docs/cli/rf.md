# Transforming Robot Framework Output

Transforms and inserts Robot Framework data from an `output.xml` file into the database.

```shell
cetk-lt [global options] rf <path> <job> <db_name> [options]
```

After a successful run, cetk-lt prints the normalized UTC `test_run_starttime`
to stdout (e.g. `2024-01-01 01:01:01.111000Z`), which can be
[piped](../piping.md) into a subsequent `cetk-lt log` call.

## Arguments

### `path`
Path to the Robot Framework `output.xml` source file — requires [output.xml schema version 5](#rf-compatibility).

### `job`
Name of the job that was tested. Used to identify the test run in the database.

### `db_name`
Name of the target database.

## Options

### `--help` / `-h`
Show help message and exit.

---

### `--test-run-starttime` / `-t` (str) { #test-run-starttime }
Start timestamp of the test run.

- Format: `%Y-%m-%d %H:%M:%S.%f` with optional timezone suffix `Z` or `+/-HH:MM`
- Default: auto-detected from the first suite in `output.xml`

!!! note
    If omitted, cetk-lt reads the start timestamp from the first suite found in
    `output.xml`. The detected timestamp is normalized to UTC and printed to stdout.

    See [Timestamp & Timezone](../timestamp.md) for accepted formats and timezone handling.

### `--log-link` (str)
URL to the Robot Framework `log.html` artifact — stored in the database and
used to link back to the RF log from visualization dashboards.

- Default: `""` (no link stored)

### `--report-link` (str)
URL to the Robot Framework `report.html` artifact — stored in the database and
used to link back to the RF report from visualization dashboards.

- Default: `""` (no link stored)

## RF Compatibility

cetk-lt supports **output.xml schema version 5**, introduced with Robot Framework 7.0.

!!! note
    output.xml files generated by Robot Framework < 7.0 use an earlier schema version
    and are not supported.

| RF version | Status        |
|------------|---------------|
| 7.x        | Supported     |
| 8.x        | Not validated |
| < 7.0      | Not supported |
