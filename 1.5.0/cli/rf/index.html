    Transforming Robot Framework Output¶

    Transforms and inserts Robot Framework data from an output.xml file into the database.

    cetk-lt [global options] rf <path> <job> <db_name> [options]

    After a successful run, cetk-lt prints the normalized UTC test_run_starttime to stdout (e.g. 2024-01-01 01:01:01.111000Z), which can be piped into a subsequent cetk-lt log call.

    Arguments¶

    path¶

    Path to the Robot Framework output.xml source file — requires output.xml schema version 5.

    job¶

    Name of the job that was tested. Used to identify the test run in the database.

    db_name¶

    Name of the target database.

    Options¶

    --help / -h¶

    Show help message and exit.

    --test-run-starttime / -t (str)¶

    Start timestamp of the test run.

    • Format: %Y-%m-%d %H:%M:%S.%f with optional timezone suffix Z or +/-HH:MM
    • Default: auto-detected from the first suite in output.xml
    • Environment variable: CETK_LT_RF_TEST_RUN_STARTTIME

    Note

    If omitted, cetk-lt reads the start timestamp from the first suite found in output.xml. The detected timestamp is normalized to UTC and printed to stdout.

    See Timestamp & Timezone for accepted formats and timezone handling.

    --test-run-starttime-naive-tz (str)¶

    Timezone to assume for the test_run_starttime value — the single timestamp that identifies the test run in the database.

    • Accepted formats: IANA name (e.g. Europe/Berlin) or fixed offset (e.g. +01:00) — see Timestamp & Timezone for DST behavior

    • Default: None (UTC assumed if no timezone info is present)

    • Environment variable: CETK_LT_RF_TEST_RUN_STARTTIME_NAIVE_TZ

    --source-log-naive-tz (str)¶

    Timezone to assume for the individual event timestamps inside the source file.

    • Accepted formats: IANA name (e.g. Europe/Berlin) or fixed offset (e.g. +01:00) — see Timestamp & Timezone for DST behavior

    • Default: None (UTC assumed if no timezone info is present)

    • Environment variable: CETK_LT_RF_SOURCE_LOG_NAIVE_TZ

    Difference between the two timezone options
    • --test-run-starttime-naive-tz — applies to the test run identifier: one timestamp per run, used as the grouping key in the database
    • --source-log-naive-tz — applies to every event timestamp inside the source log file (e.g. keywords/suites in RF XML, individual lines in a log source)

    Both options are independent and can carry different timezones — for example when the CI server runs in UTC but the source log file was generated on a Europe/Berlin machine.

    --host (str)¶

    IP address or DNS name of the database host.

    • Default: "" (local socket)
    • Environment variable: CETK_LT_DB_HOST

    --port (int)¶

    Port of the database.

    • Default: 5432
    • Environment variable: CETK_LT_DB_PORT

    --username / -u (str)¶

    Username for the database login.

    • Default: ""
    • Environment variable: CETK_LT_DB_USER_NAME

    --password / -p (str)¶

    Password for the database login.

    • Default: ""
    • Environment variable: CETK_LT_DB_USER_PW

    --log-link (str)¶

    URL to the Robot Framework log.html artifact — stored in the database and used to link back to the RF log from visualization dashboards.

    • Default: "" (no link stored)
    • Environment variable: CETK_LT_RF_LOG_LINK

    --report-link (str)¶

    URL to the Robot Framework report.html artifact — stored in the database and used to link back to the RF report from visualization dashboards.

    • Default: "" (no link stored)
    • Environment variable: CETK_LT_RF_REPORT_LINK
    Exit Codes

    See Command Line Interface for the complete exit code reference.

    RF Compatibility¶

    cetk-lt supports output.xml schema version 5, introduced with Robot Framework 7.0.

    Note

    output.xml files generated by Robot Framework < 7.0 use an earlier schema version and are not supported.

    RF version Status
    7.x Supported
    8.x Not validated
    < 7.0 Not supported
